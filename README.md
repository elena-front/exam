Написать для себя заготовку под экзамен (её прямо можно будет использовать на экзамене):
- работающий сервер на Express, архитектура - REST

- декомпозиция на сервере - сервисы, контроллеры, роуты

- две сущности в БД:
a) пользователь (User): имя, почта, пароль - это минимум
б) любая на ваш выбор - посты, задачи, товары, компании, но - с привязкой к пользователю.

- для пользователей - эндпоинты для регистрации, входа, выхода и обновления токенов (продления сессии)

- для второй сущности - полный набор CRUD-операций (получение всех, получение по ID, создание, удаление, редактирование)

- клиент - сборка через Vite, фреймворк - React

- архитектура - на ваше усмотрение

- связь клиента и сервера

- работающая JWT-авторизация и аутентификация, пользователь регистрируется, заходит, выходит, не слетает при обновлении страницы

- хорошо если будет закрыта уязвимость IDOR - и пользователь не сможет удалить/отредактировать задачи/посты/товары, созданные другим пользователем
